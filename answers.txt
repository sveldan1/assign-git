•  What does fast-forward in git mean?
   Git fast forward is said to happen when you merge a branch that is ahead of the branch with head pointer i.e. your current branch.
•  Why should you have different branches?
   For a team to collaborate and work on the same project without making any changes to the main branch, different branches should be present.
•  In a project how many remote Git repositories do you usually have?
   Depending on the size of the project, number of remote Git repositories vary. For instance, if a project involves UI, backend and various such components, we create separate repositories for each of them.
